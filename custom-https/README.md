## Overview

**SAP CI HTTPS Extension Adapter** is a future open-source research initiative focused on exploring extension capabilities around HTTPS-based integration scenarios in **SAP BTP Integration Suite / SAP Cloud Integration**.

This project is currently in the **conceptual alignment and architectural structuring phase**.

The purpose is not to replace the SAP standard HTTPS adapter.

The purpose is to investigate how selected extension capabilities, runtime patterns, governance concepts, observability mechanisms, and reusable integration behaviors can be designed around HTTPS-based flows in a clean, controlled, and community-reviewable way.

## Why This Exists

HTTPS remains one of the most widely used integration protocols in SAP Cloud Integration landscapes.

However, enterprise HTTP-based integration often requires more than simple request/response connectivity.

Future exploration areas may include:

* governance-oriented runtime behavior;
* reusable extension patterns;
* improved operational visibility;
* domain-aligned HTTP integration design;
* structured diagnostics;
* metadata-driven processing concepts;
* cleaner separation between connectivity and orchestration;
* community-reviewable implementation patterns.

The final scope will be defined carefully.

No unsupported technical claim is made at this stage.
